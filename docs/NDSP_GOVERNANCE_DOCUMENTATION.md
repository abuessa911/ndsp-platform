## 🎯 تعريف النظام الرسمي

**NDSP** هو اختصار لـ:
- **Non-Discretionary Support Platform**
- **نظام دعم القرار المالي التحليلي**

### التعريف الكامل:
**NDSP ليس نظام تنفيذ صفقات تلقائي. NDSP هو منصة تحليل ودعم قرار مالي تقدم:**
- قراءة منظمة للسوق
- تحليل منهجي للطبقات المؤثرة
- سيناريوهات مرجعية واضحة
- مستويات قرار محددة
- توجيه تحليلي شامل

## ⚠️ إخلاء المسؤولية الحوكمي الكامل

```
════════════════════════════════════════════════════════════════════
NDSP - نظام دعم القرار المالي

إخلاء مسؤولية شامل:

1. NDSP هو نظام تحليل وعرض معلومات فقط
2. NDSP لا يقدم توصيات مالية مباشرة
3. NDSP لا ينفذ أوامر تداول
4. NDSP لا يضمن النتائج
5. NDSP لا يدير أموال المستخدم
6. NDSP لا يربط حسابات تداول حقيقية
7. المستخدم مسؤول بالكامل عن قراراته المالية

جميع الخدمات مقدمة "كما هي" بدون ضمانات من أي نوع.
════════════════════════════════════════════════════════════════════
```

## 📚 الطبقات التحليلية الـ 10

### الطبقة 1: TDL - تحليل الاتجاه اليومي
```
الوصف: قراءة الاتجاه الأساسي للسعر على مدار الجلسة اليومية
المخرجات المسموح بها:
  • الاتجاه الحالي (صاعد/هابط/جانبي)
  • قوة الاتجاه (1-10)
  • مناطق الدعم الأساسي
  • مناطق المقاومة الأساسية
  • آفق الاتجاه (ساعات)

ممنوع تماماً:
  • إصدار توصية شراء أو بيع مباشرة
  • ضمان الاتجاه
  • توقع سعر دقيق
```

### الطبقة 2: NMP - نقاط التحول الهامة
```
الوصف: تحديد المستويات الحرجة التي قد يتغير عندها الاتجاه
المخرجات المسموح بها:
  • المستويات الحرجة الأولى والثانية
  • احتمال كسر كل مستوى (%)
  • منطقة عدم الاستقرار
  • منطقة التأرجح المحتملة

ممنوع تماماً:
  • الادعاء بأن الكسر سيحدث مؤكداً
  • ربط الكسر بـ "ربح مضمون"
  • إصدار أمر شراء/بيع عند المستوى
```

### الطبقة 3: Macro Layer - الطبقة الكلية
```
الوصف: تأثير البيانات الاقتصادية والأخبار الكبرى
المخرجات المسموح بها:
  • بيئة السوق الكلية (موات/غير موات)
  • آخر بيانات اقتصادية مهمة
  • التأثير المتوقع على الأصل
  • مخاطر السياسة النقدية
  • تقويم الأحداث الاقتصادية القادمة

ممنوع تماماً:
  • التنبؤ الدقيق بأرقام اقتصادية
  • ضمان تأثير الأخبار
  • إصدار توصيات بناءً على بيانات اقتصادية فقط
```

### الطبقة 4: Day Logic - منطق اليوم
```
الوصف: التحليل المنطقي لحركة السعر خلال جلسة اليوم
المخرجات المسموح بها:
  • نمط اليوم (تصحيح/عكس/استمرار)
  • منطقة التجميع الحالية
  • القوة النسبية للشراء والبيع
  • الحجم النسبي
  • السلوك المتوقع

ممنوع تماماً:
  • ضمان النمط سيستمر
  • توقع السعر الدقيق
  • إصدار أوامر بناءً على المنطق فقط
```

### الطبقة 5: Scenario Levels - مستويات السيناريو المرجعية
```
الوصف: المستويات المرجعية لكل سيناريو محتمل
المخرجات المسموح بها:
  • منطقة تفعيل السيناريو
  • الهدف المرجعي (ليس التنبؤ)
  • مستوى إلغاء السيناريو
  • احتمال كل سيناريو (%)
  • الإطار الزمني لكل سيناريو

ممنوع تماماً:
  • الادعاء بأن الهدف "سيصل 100%"
  • ضمان تحقق السيناريو
  • إصدار أمر شراء/بيع على أساس السيناريو
  • قول "استثمر بناءً على هذا السيناريو"
```

### الطبقة 6: Decision State - حالة القرار
```
الوصف: التصنيف الكامل لحالة القرار الحالية
المخرجات المسموح بها:
  • حالة القرار: واضح/يحتاج تأكيد/غير مكتمل
  • ما ينقص للقرار الكامل
  • مستوى الوضوح (%)
  • الإطار الزمني للقرار
  • نقاط الانتظار المهمة

ممنوع تماماً:
  • قول "القرار نهائي وملزم"
  • فرض قرار على المستخدم
  • ضمان صحة القرار
  • قول "تنفيذ فوري"
```

### الطبقة 7: Quality Decision - جودة القرار
```
الوصف: تقييم موثوقية ودقة القراءة التحليلية
المخرجات المسموح بها:
  • درجة الثقة (0-100%)
  • مصادر البيانات المستخدمة
  • مستوى الاتفاق بين المؤشرات
  • العوامل الداعمة
  • العوامل المعارضة
  • التوصية بانتظار مزيد من التأكيد

ممنوع تماماً:
  • قول "الثقة 100% - تنفيذ الآن"
  • ضمان دقة التحليل
  • ادعاء أن القراءة "صحيحة 100%"
```

### الطبقة 8: Market Reading - قراءة السوق
```
الوصف: الفهم الشامل لحالة السوق الحالية
المخرجات المسموح بها:
  • الاتجاه الأساسي
  • الزخم الحالي
  • التوازن الحالي (عرض/طلب)
  • الحالة النفسية (تفاؤل/تشاؤم/محايد)
  • مستويات المشاركة
  • آفق القراءة

ممنوع تماماً:
  • ادعاء القراءة "خالية من الخطأ"
  • فرض فهم واحد للسوق
  • ضمان استمرار القراءة
  • قول "القراءة = أمر تنفيذ"
```

### الطبقة 9: Risk Notes - ملاحظات المخاطر
```
الوصف: تحديد المخاطر المحتملة والمؤثرة
المخرجات المسموح بها:
  • المخاطر الرئيسية والثانوية
  • مستوى المخاطر الكلي
  • مسافة الأمان المقترحة
  • إدارة التعريض
  • السيناريوهات المخاطرة
  • تحذيرات من أحداث محتملة

ممنوع تماماً:
  • قول "لا توجد مخاطر"
  • ضمان إدارة المخاطر
  • ادعاء أن المسافة الموصى بها "آمنة 100%"
  • قول "خطر منخفض = شراء آمن"
```

### الطبقة 10: Live Price Bridge - جسر السعر الحي
```
الوصف: ربط مباشر بين السعر الحي والتحليل الحالي
المخرجات المسموح بها:
  • السعر الحالي (مع آخر تحديث)
  • المسافة من مستويات حرجة
  • نقطة الدخول المرجعية (لا التنفيذ)
  • نقطة الخروج المرجعية (لا التنفيذ)
  • هل السعر قريب من منطقة حرجة
  • الوقت المتبقي للقرار

ممنوع تماماً:
  • قول "السعر وصل - تنفيذ الآن"
  • أمر تنفيذ تلقائي عند مستوى معين
  • ربط السعر بـ "أمر شراء/بيع"
  • ضمان ملائمة السعر الحالي للدخول
```

## 🔐 التنبيهات الحوكمية الإلزامية

### التنبيه الرئيسي (يجب في كل جلسة استخدام):
```
════════════════════════════════════════════════════════════════════
⚠️ إخلاء مسؤولية حوكمي إلزامي:

NDSP هو نظام تحليل ودعم قرار مالي - وليس نظام تنفيذ صفقات
✗ لا يمثل توصية مالية مباشرة
✗ لا ينفذ أوامر تداول
✗ لا يضمن النتائج
✗ لا يدير أموالك

أنت مسؤول بالكامل عن قراراتك المالية والتداولية.
════════════════════════════════════════════════════════════════════
```

### تنبيهات سياقية (يجب عند كل قرار):
```
عند عرض سيناريو:
⚠️ هذا السيناريو احتمالي فقط - ليس توقع مؤكد

عند عرض هدف سعري:
⚠️ هذا هدف مرجعي - قد لا يتحقق

عند عرض درجة ثقة:
⚠️ حتى درجة الثقة العالية لا تضمن النتيجة

عند عرض مستوى إلغاء:
⚠️ كسر المستوى ليس مؤشراً نهائياً - يحتاج تأكيد

عند طلب قرار:
⚠️ انتظر إلى أن تشعر براحة تجاه القرار
```

## 📊 جودة الحوكمة - معايير التقييم

### معايير التقييم الداخلية:

#### ✓ النظام متوافق مع الحوكمة إذا:
```
1. لا يوجد أي زر "شراء" أو "بيع"
2. لا يوجد ربط مباشر مع منصات تداول حقيقية
3. جميع الأوامر صياغتها "تحليلية" وليست "تنفيذية"
4. هناك تنبيهات واضحة في كل مكان
5. لا يوجد ضمان لأي نتيجة
6. المسؤولية تركز على المستخدم دائماً
7. كل سيناريو يحمل درجة ثقة واضحة
8. كل قرار يحمل حالة (واضح/غير مكتمل/يحتاج تأكيد)
9. لا يوجد توصيات "ملزمة"
10. النظام يشجع على الانتظار والتأكيد
```

#### ✗ النظام مخالف للحوكمة إذا:
```
1. ✗ وجود أزرار "شراء/بيع" مباشرة
2. ✗ ربط مع وسيط حقيقي
3. ✗ تنفيذ أوامر تلقائية
4. ✗ توصيات بدون درجة ثقة
5. ✗ ضمان النتائج
6. ✗ إدارة أموال حقيقية
7. ✗ إخفاء المخاطر
8. ✗ فرض قرار على المستخدم
9. ✗ عدم وجود تنبيهات
10. ✗ ادعاء "دخول مضمون" أو "ربح مضمون"
```

```
اقرأ جميع الطبقات التحليلية الـ 10

حدد نقاط القوة والضعف في القراءة
```

#### الخطوة 3: تقييم حالة القرار
```
تحقق من درجة الثقة
ادرس ما ينقص للقرار الكامل

```

```
اقرأ ملاحظات المخاطر كاملة
افهم السيناريوهات المخاطرة
حدد مسافة الأمان المناسبة لك
```

#### الخطوة 5: قرار مستقل
```
اتخذ قرارك الخاص بناءً على فهمك
لا تعتمد على NDSP وحده
استشر مصادر إضافية
أنت المسؤول الوحيد
```

## 📞 الدعم والاستفسارات

### للإبلاغ عن مشاكل حوكمية:
```
إذا وجدت أي نص يخالف معايير الحوكمة:
1. وثق المكان بالضبط
2. اذكر الصياغة الممنوعة
3. ارسل تقرير مفصل
4. لا تستخدم هذا النص في قراراتك
```

### استفسارات الخصوصية:
```
NDSP لا يجمع بيانات شخصية
NDSP لا يحفظ سجل المستخدمين
NDSP لا يرسل بيانات إلى جهات خارجية
جميع البيانات تبقى محلية على جهازك
```

## ⚖️ المسؤولية القانونية

### شروط الاستخدام:

```
باستخدامك لـ NDSP:
1. أنت توافق على أنك فهمت طبيعة النظام
2. أنت تقبل المسؤولية الكاملة عن قراراتك
3. أنت تقر بأن NDSP لا يقدم ضمانات
4. أنت تتحمل جميع الخسائر المحتملة
5. أنت لن تحمل مطورو NDSP أي مسؤولية قانونية
6. أنت ستستخدم NDSP بمسؤولية
7. أنت ستتقيد بجميع القوانين المحلية
```

## حوكمة التطوير والاختبار: منع الحقن البرمجي حتى لأغراض الاختبار

يحظر في منصة NDSP استخدام أي سكربتات حقن أو ترقيع على ملفات الواجهة المنشورة، سواء في بيئة الإنتاج أو لغرض الاختبار المؤقت، إذا كان السكربت يقوم بأي من الآتي:

- حقن JavaScript أو CSS داخل صفحات الإنتاج بعد النشر.
- تعديل سلوك الواجهة من خارج مصدرها الرسمي.
- تجاوز بنية التطبيق الأصلية عبر سكربتات منفصلة غير معتمدة.
- إضافة روابط، فلاتر، بيانات، أو حالات عرض عبر ملفات مؤقتة مركبة فوق الصفحة.
- استخدام Console snippets أو runtime patches أو override scripts لاختبار وظائف مرتبطة بالباك إند.
- إضافة بيانات بديلة أو fallback غير صادر من الباك إند.
- تعديل ملفات `/var/www` كبديل عن تعديل المصدر الرسمي وبناء نسخة إنتاجية جديدة.
- اعتماد أي نتيجة اختبار ناتجة عن سكربت حقن كدليل جاهزية إنتاجية.

الاختبار المقبول يجب أن يتم فقط عبر واحد من المسارات التالية:

1. تعديل المصدر الرسمي للواجهة ثم بناء نسخة جديدة.
2. استخدام بيئة Staging منفصلة عن الإنتاج.
3. استخدام Feature Branch مع مراجعة قبل الدمج.
4. اختبار API مباشرة عبر طلبات قراءة فقط دون تعديل الواجهة.
5. استخدام أدوات مراقبة أو تقارير لا تغيّر الصفحة ولا تضيف سلوكاً جديداً.
6. اختبار عقود API بين الواجهة والباك إند دون حقن أو تجاوز لمسار التطبيق الرسمي.

أي تغيير في الواجهة يجب أن يكون جزءاً من الكود المصدري الرسمي، ومربوطاً بالباك إند عبر عقود API واضحة، وقابلاً للمراجعة والتتبع والرجوع عنه من خلال نظام النسخ أو المستودع.

لا يجوز اعتماد أي نتيجة اختبار ناتجة عن سكربت حقن كدليل جاهزية إنتاجية. تعتبر هذه النتائج مؤشرات تشخيصية فقط، ولا تدخل في اعتماد الإطلاق.

قاعدة ملزمة:

> لا حقن. لا ترقيع فوق الإنتاج. لا اختبار بسكربتات مؤقتة على الواجهة المنشورة.  
> المصدر الرسمي + الباك إند + بيئة اختبار معزولة فقط.
