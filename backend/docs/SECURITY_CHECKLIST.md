# NDSP Backend — قائمة الأمان الإلزامية (SECURITY_CHECKLIST)

راجع هذه القائمة قبل أي إطلاق. الحوكمة والأمان فوق كل شيء.

## 1) منع كشف الأسرار
- [ ] لا توجد أسرار حقيقية في أي ملف داخل المستودع (فقط `.env.example` بأسماء المتغيّرات).
- [ ] `.env` مُدرج في `.gitignore` ولم يُرفع أبدًا إلى Git.
- [ ] الأسرار (DB, JWT, FERNET, مفاتيح المزوّد/الدفع، توكن تيليجرام، SMTP) في `.env` على الخادم فقط.
- [ ] `verify_ndsp_backend.sh` يمرّ دون اكتشاف قيم حقيقية في `.env.example`.

## 2) JWT
- [ ] `JWT_SECRET` قوي وعشوائي (≥ 32 بايت)، مولّد لكل بيئة على حدة.
- [ ] فصل access (قصير) عن refresh (أطول)؛ التحقّق من `type` في كل مسار.
- [ ] لا توضع بيانات حسّاسة داخل الـ payload (فقط `sub`, `role`, `type`, `iat`, `exp`).

## 3) كلمات المرور
- [ ] التجزئة بـ **Argon2** (مع دعم التحقّق من bcrypt للقديم) — لا تخزين نصّي إطلاقًا.
- [ ] حد أدنى لطول كلمة مرور الأدمن (≥ 10) في `bootstrap`.

## 4) صلاحيات الأدمن من قاعدة البيانات
- [ ] صلاحية الأدمن تُقرأ من عمود `users.role` — **ليست** من التوكن وحده.
- [ ] `require_admin` يرفض أي دور غير `admin` بـ 403.
- [ ] لوحة الأدمن على نطاق منفصل `admin.ndsp.app` مع وصول شبكي مقيّد.

## 5) CORS — الدومينات الرسمية فقط
- [ ] `CORS_ORIGINS=https://ndsp.app,https://my.ndsp.app,https://admin.ndsp.app` فقط.
- [ ] لا `*` ولا نطاقات تطوير في الإنتاج.

## 6) Rate limiting
- [ ] حدّ عام لكل IP (`RATE_LIMIT_PER_MINUTE`) وحدّ أصرم على المصادقة (`AUTH_RATE_LIMIT_PER_MINUTE`).
- [ ] التسجيل والدخول والدفع محميّة بتحديد المعدّل.

## 7) Audit logs
- [ ] كل إجراء إداري/دفع/تفعيل يُسجَّل في `audit_logs` (الفاعل، الإجراء، الهدف، وقت).
- [ ] `meta` مقنّعة فقط — **لا أسرار، لا بصمات خام** (IP يُخزَّن مجزّأً).

## 8) Safe errors
- [ ] لا تُكشف آثار التتبّع (stack traces) للعميل؛ المعالج العام يعيد خطأً عامًا `INTERNAL_ERROR`.
- [ ] مغلّف خطأ موحّد `{ error: { code, message_ar, message_en } }` فقط.
- [ ] الوثائق التفاعلية (Swagger/redoc) **معطّلة** في الإنتاج (`docs_url=None`).

## 9) حماية Webhooks
- [ ] webhook الدفع يتحقّق من توقيع المزوّد (`x-nowpayments-sig` عبر HMAC-SHA512) قبل الثقة.
- [ ] لا تفعيل اشتراك إلا بعد تحقّق ناجح أو اعتماد أدمن. **لا تفعيل تلقائي**.

## 10) منع أسرار داخل الـ frontend
- [ ] الواجهة عرض فقط؛ لا مفاتيح/توكنات في كود الواجهة.
- [ ] ربط تيليجرام عبر **رمز لمرة واحدة**؛ توكن البوت لا يُعاد للعميل.
- [ ] `chat_id` يُخزَّن مجزّأً (`chat_id_hash`)، لا خام.

## 11) المنقّح والحوكمة (إلزامي)
- [ ] كل مخرجة عامة (قرار/سيناريو/موجز/لوحة) تمرّ من `sanitize_decision`.
- [ ] القائمة السوداء تمنع: معادلات/أوزان/تسجيل داخلي/أسماء طبقات مخفية/ترتيبها/بصمات خام/شراء/بيع/الآن/دخول/جني/وقف/تنفيذ/هدف مضمون/«نصيحة مالية».
- [ ] `/admin/layers` يعيد فقط الأسماء الأربعة + `hidden_count` — لا أكثر.
- [ ] «Nawaf Golden Alignment» دائمًا مرفقة بنصّ «معزّز سياقي للجودة، لا ضمان».
- [ ] اختبار آلي يفشل لو ظهر أي مفتاح ممنوع في مخرجة عامة.

## 12) اللغات
- [ ] العبرية محجوبة على مستوى الـ API (حارس `Accept-Language` يرفض `he/iw/he-IL` بـ 400).
- [ ] اللغات المدعومة: العربية والإنجليزية فقط.

## 13) النسخ والتقارير
- [ ] كل تركيب/تعديل عبر السكربت يأخذ نسخة احتياطية في `/home/nawaf511/ndsp_backups` وينتج تقريرًا في `/home/nawaf511/ndsp_launch_reports`.
- [ ] لا تُكتب تقارير داخل المشروع، ولا تُنشأ مسارات بديلة.

## 14) عام
- [ ] HTTPS إلزامي + شهادات سارية (Let's Encrypt).
- [ ] رؤوس أمان على nginx (HSTS, CSP عند الإمكان).
- [ ] قاعدة البيانات لا تستمع علنًا (localhost أو شبكة خاصة).
- [ ] تحديثات الاعتماديات بانتظام.
